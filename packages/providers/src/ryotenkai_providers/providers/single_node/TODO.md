## GPU Resource Contention (ожидание при занятой GPU)

При batch-запуске нескольких конфигов параллельно на single node,
GPU может быть занята другим раном или инференсом.

**Необходимо реализовать:**

1. **Проверку доступности GPU перед стартом тренировки**
   - `nvidia-smi` → проверить используемую VRAM
   - Если VRAM загружена выше порога (например 80%) — не стартовать

2. **Режим ожидания (wait-for-gpu)**
   - Если GPU занята — не падать сразу, а встать в очередь
   - Поллить `nvidia-smi` каждые N секунд
   - Таймаут на ожидание (например 30 мин), после которого — fail
   - Логирование: `"GPU busy (12.3/15.6 GB used), waiting..."`

3. **Простой file-based lock**
   - Один тренировочный процесс = один lock-файл на GPU
   - Второй процесс видит lock → ждёт его снятия
   - Защита от zombie-locks (проверка PID в lock-файле)

**Приоритет**: нужно для batch smoke-тестирования на локальной машине.
Без этого параллельные раны на single node будут конфликтовать за GPU.
