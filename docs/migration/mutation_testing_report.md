**Scope:** 3 of 10 hotspot modules ŌĆö chosen by ascending size to validate the

**Lane status:** GREEN throughout. Cosmic Ray reverts each mutation after the

## Goal

Validate that the test suite that emerged after Phases 2-5 of mock elimination
actually **catches bugs** ŌĆö not just that "tests run". Mutation testing
introduces small, semantically-meaningful edits to production code and asks
whether the test suite flags them. A surviving mutation = a behavior change
no test would notice = either a missing test, an over-mocked test, or genuinely
dead production code.

### A large fraction of "survivors" are false positives

Cosmic Ray includes operator families that mutate the `|` bitwise-or token,
but PEP 604 type unions also use `|` (e.g. `int | None`, `IGPUProvider | None`).
Mutations to those expressions are no-ops at runtime ŌĆö Python doesn't evaluate
annotations for behavior. After filtering type-annotation noise:

| module | total survived | BitOr-on-annotations (noise) | **real signal** |
|---|---:|---:|---:|
| `event_bus.py` | 162 | ~88 | **~74** |
| `deployment_manager.py` | 13 | 11 | **2** |
| `mlflow_relay.py` | 93 | 22 | **71** |

Effective kill rates after subtracting noise:

- `event_bus.py`: 74 real survivors / (328 ŌłÆ 88 noise) = **~69%** real kill rate
- `deployment_manager.py`: 2 real survivors / (15 ŌłÆ 11) = **~50%** real kill rate
- `mlflow_relay.py`: 71 real survivors / (198 ŌłÆ 22) = **~60%** real kill rate

**Aggregate effective kill rate (real signal only): ~64%.**

This is on the boundary of "tests are useful but have gaps" (50-70%) ŌĆö solidly
above "low signal" but below the "industry-strong" 80%+ threshold.

## What the survivors tell us, by category

### Genuine test gaps (priority: fix tests)
- **Per-kind dispatch in `mlflow_relay._forward`** (mutations 3, 8). Add a
  parametrised test that publishes one event per kind and asserts the matching
  mock-client method was called. Today's tests appear to be over-mocked: they
  verify "client interaction" but not which method.
- **Disk-pressure rate limiter in `event_bus._signal_disk_pressure`** (mutations 1, 2).
  Add a test that publishes failures twice within 60s and asserts only one
  WARN log fires. Mock `logging.getLogger(__name__).warning` and assert the
  call count.
- **Circuit-breaker boundary** (mutation 4). Add a test where `now -
  opened_at == active_cooldown` exactly ŌĆö should return `False`.

### Default-value drift (priority: medium)
- `mlflow_relay.MLflowRelayCircuitBreaker` field defaults (mutation 6) and
  `event_bus` module constants (mutation 9) are unpinned. Either add explicit
  pin tests or treat them as configuration that should live in
  `ryotenkai_shared.config`.

### Cosmic Ray noise (priority: filter in CI config)
- BitOr mutations on PEP 604 type unions account for ~22-88 false positives
  per module. Future CI integration should suppress `core/ReplaceBinaryOperator_BitOr_*`
  operators on lines containing `|` in annotation position. Cosmic Ray's
  `excluded-modules` and per-operator opt-out can do this with a
  `filters` block; alternatively, switch to **`mutmut`** which is
  type-annotation-aware.

### Deliberately-untested branches (priority: low ŌĆö leave alone)
- `mlflow_relay.py:303-309` is marked `# pragma: no cover`. Tell Cosmic Ray
  to skip these lines (or accept them as a known small noise floor).

## CI integration recommendation

**Yes ŌĆö add Cosmic Ray to nightly CI.** Justification:

1. Real kill rate (~64% after filtering type-annotation noise) is above the
   30% "low signal ŌĆö don't bother" threshold.
2. Cost is bounded (~2 hours total for all 10 hotspots).
3. Each surviving mutation produces actionable signal: file:line + the exact
   mutation type, so a developer can write a test in minutes.

Concrete proposal:

- **Nightly job** runs the 5 modules in `scripts/run_mutation_testing.sh::TARGETS`
  (event_bus, deployment_manager, mlflow_relay, pod_terminator, supervisor)
  with `FAST=0`, archives `sessions/*.sqlite` and `sessions/*_report.html`
  as build artifacts.
- **Weekly job** extends to the full top-10 hotspot set.
- **Regression threshold**: fail the nightly if any module's kill rate drops
  by more than **5 percentage points** vs the previous run. (Stored as
  `sessions/SUMMARY.md` ŌĆö diff that file in CI.)
- **Filter PEP 604 noise**: add a Cosmic Ray operator-filter or migrate to
  `mutmut` before promoting from advisory to blocking.

The mutation-testing job MUST remain advisory-only until the type-annotation
filter is in place ŌĆö otherwise the false-positive rate (BitOr on unions)
will dominate the signal and developers will tune it out.
