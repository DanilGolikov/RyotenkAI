## Suggested batch plan for downstream conversion

Ordered by leverage (count × ease). Each batch ≈ 200–400 hits.

1. **Batch 1: bare `MagicMock()` → `SimpleNamespace`** — ~1627 hits — Mechanical. Group by file; each replacement is local.
2. **Batch 3: `MagicMock(return_value=...)` → tiny Fake or factory** — ~349 hits — Mechanical-ish. Replace per-file with a small dataclass.
3. **Batch 4: `patch.dict(os.environ, ...)` → `monkeypatch.setenv`** — ~68 hits — Mechanical. Pure rewrite.
4. **Batch 5: `MagicMock(spec=ConcreteClass)` → Fake or real construct** — ~22 hits — Judgment per class. Build Fakes in tests/_fakes.
5. **Batch 6: `AsyncMock` → Fake async client** — ~82 hits — Judgment. Group by client interface.
6. **Batch 7: `@patch` on external libs (time/subprocess/etc)** — ~274 hits — Acceptable to keep; or migrate to ManualClock / fixtures.
7. **Batch 8: `@patch` internal private fns / Protocols (REFACTOR)** — ~330 hits — Requires production dep-injection. Smaller per change but more risk.
8. **Batch 9: `patch.object(...)` cleanup** — ~352 hits — Judgment. Replace with constructor injection or Fake.
