# adapter_cache.revision — отложенная идея

## Суть

Опциональное поле `revision` в `AdapterCacheConfig` позволяет пользователю явно указать
ревизию HF-адаптера вместо автовычисленного тега по fingerprint датасета.

## Два режима значения

- **Тег** (`phase-0-sft-dsABC...`) — загружает последний адаптер с этим fingerprint (тег подвижный, двигается при каждом re-upload)
- **Commit SHA** (`abc123def4...`) — загружает точно этот коммит, неизменно

## Поведение

- Если задан `revision` и `upstream_retrained=True` — explicit revision выигрывает, каскад игнорируется, логируется STRONG WARNING о потенциальной несовместимости с новым upstream-адаптером
- Если задан `revision` и он не совпадает с auto-fingerprint тегом — WARNING о расхождении с текущим датасетом
- Если задан `revision` и тег/SHA не найден на HF — hard-fail (в отличие от auto-режима где 404 → train)

## Почему отложено

Сложный edge case, нужен только для продвинутых пользователей (внешние адаптеры, нестандартные теги).
99% сценариев покрываются авто-режимом по fingerprint.
